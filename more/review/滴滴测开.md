1.String、StringBuilder和 StringBuffer 的区别是什么？
String：
- 是不可变的，一旦被创建，它的值就不可被改变，每次对String进行修改（拼接、替换）实际上都是创建一个新的String对象。
- 因为其不可变，所以是线程安全的
- 因为其不可变，在对字符串进行频繁修改时会产生大量临时对象从而影响性能
StringBuffer:
- 是可变的，可以在其原始对象上进行修改，而不是创建新的对象
- 是线程安全的，他的大多数方法都被sync修饰
- 低性能。同步锁带来额外开销
StringBuilder:
- 可变，内部的字符数组可动态扩容
- 线程不安全，方法没有同步锁
- 高性能，无锁开销，适用于单线程

1.HashMap和ConcurrentHashMap的区别
线程安全性：HashMap线程不安全；后者线程安全
底层数据结构：HashMap是数组+链表+红黑树；ConcurrentHashMap是Node数组+链表/红黑树，并且使用了CAS和synchronized。
是否允许null值：前者允许null值和null键；后者不允许

4.redis：
数据类型：String，hash、list、set、zset、bitmap。key必须是string
- 持久化：快照，在特定时间点创建整个二进制快照文件，耗时、易丢失。    日志：记录每次写命令。回复的速度慢
- redis快的原因：1.数据存储在内存中 2.单线程
- 穿透：一直访问一个不存在的数据，导致请求直接打到数据库中
- 击穿：一个热点的key过期瞬间大量请求涌入击穿缓存，打到数据库
- 雪崩：同一时间大量key集中过期，导致所有请求涌向数据库

5.查询平均成绩>80的学生姓名：
select s.name,avg(sc.grade)
from student s 
join score sc on s.id=sc.student_id
group by avg(grade)>80
order by avg(sc.grade)desc

select s.name from student s 
where s.id in (
select student_id from score  
group by student_id
having avg(grade)>80
)
